# 任务 02 —— 坐实/降级 C1：NFE 漂移对照 + 1.4B scaling 决策

> 风险：C1"一步=去字符漂移"机制未隔离(仅 teacher-MS 63.78 vs B 65.99 一对，混淆"步数"与"teacher/学生")；1.4B 放大当前是负证据(误跑 FM、det 66.95<0.5B；teacher 从零训 CharAcc~29)。详 `../story_evidence_audit.md` C1。

## Part A —— NFE 漂移隔离对照（坐实机制，C1 想保留机制则必做）
- **目的**：把"多步 vs 一步"单独隔离出来。
- **设计**：**同一个模型**测不同 NFE 的 CharAcc——
  - 选项1(最干净)：取 teacher，固定权重，推理 NFE∈{1, 4, 8, 28}，看 CharAcc 是否随步数升高而**下降**(=多步采样字符漂移)。
  - 选项2(补充)：取一个蒸馏学生(B 或 2a)，NFE=1 vs 强行多步，对比。
  - 选项3(质化)：可视化同一样本多步采样中途的字符抖动/漂移帧，做 figure。
- **判据**：若 NFE 升、CharAcc 降 → "一步消除漂移"成立，C1 机制坐实；否则 C1 只讲效率、删"漂移"措辞。
- **工作量**：纯推理+评测，~半天(一个模型 ×4 个 NFE × 847)。

## Part B —— 1.4B scaling 决策（先拍板，再决定跑不跑）
- **现状**：1.4B 蒸馏被查实跑错(FM 而非 shortcut distill)，teacher 从零训配置疏漏、CharAcc~29 平台。当前数据**不能进正文**(反打)。
- **三选一(先问用户/主导)**：
  1. **修好重做**：teacher 从 `VOSR_1.4B_ms` 预训练 init(非从零) → 正确 `train_vosr_distill.py` + `distill_type: shortcut`(与 0.5B 同口径) → 蒸馏 → 评测。算力大(226.31)。
  2. **降级为趋势**：只在正文写"框架可扩展"定性一句，不放主表数字，附录注明口径。
  3. **直接砍**：正文不提 scaling，省事，0.5B 足以支撑 story。
- **建议**：时间紧 → 选 3 或 2；若想要"可放大"卖点且有算力 → 选 1，但必须同口径(shortcut)。
- **工作量**：选1 = teacher 重训(数天)+蒸馏(~14h)+评测；选2/3 = 0。
