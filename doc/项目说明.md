# VOSR 项目说明与操作文档

本文档基于当前仓库源码整理，说明 VOSR 项目的整体作用、目录结构、核心模块职责、训练/蒸馏/推理流程以及常用配置方法。

## 1. 项目概览

VOSR 是一个面向图像超分辨率的视觉生成模型项目，全称为 **A Vision-Only Generative Model for Image Super-Resolution**。项目核心思想是：不使用文本条件，而是使用低质量图像的 VAE latent 与 DINOv2 视觉特征作为条件，通过 DiT/flow matching 生成更高质量的超分图像。

项目支持两类模型：

- **multi-step 模型**：使用多步 flow matching 采样，默认推理步数为 25，质量和生成细节更稳定。
- **one-step 模型**：由 multi-step teacher 蒸馏得到，默认 1 步推理，速度更快。

项目支持两种规模：

- **VOSR-0.5B**：使用 SD2.1 VAE，latent 通道为 4，推理时可使用轻量 decoder。
- **VOSR-1.4B**：使用 Qwen-Image 2D VAE，latent 通道为 16，细节保真能力更强。

## 3. 环境安装

建议使用 Python 3.8 或以上版本。README 中给出的方式如下：

```bash
conda create -n vosr python=3.8
conda activate vosr
pip install -r requirements.txt
```

`requirements.txt` 中指定了 CUDA 12.1 版本的 PyTorch：

```text
torch==2.5.1+cu121
torchvision==0.20.1+cu121
torchaudio==2.5.1+cu121
```

主要依赖包括：

- `accelerate`：分布式训练、混合精度、checkpoint 管理。
- `diffusers`：加载 SD2.1 VAE 和调度器工具。
- `timm`：Transformer 辅助模块。
- `basicsr`：RealESRGAN 退化、JPEG、滤波、噪声等工具。
- `webdataset`：读取 `.tar` shard 数据集。
- `pyiqa`：训练中评估 LPIPS、MUSIQ。
- `safetensors`：保存和加载模型权重。
- `wandb`：可选实验跟踪。

## 4. 权重目录准备

预训练权重应放在 `preset/ckpts/` 下，README 期望结构如下：

```text
preset/ckpts/
├── Qwen-Image-vae-2d/                 # Qwen-Image 2D VAE，供 1.4B 模型使用
├── stable-diffusion-2-1-base/         # SD2.1 VAE，供 0.5B 模型使用
├── sd21_lwdecoder.pth                 # SD2.1 轻量 decoder

├── VOSR_0.5B_ms/                      # 0.5B multi-step checkpoint
├── VOSR_0.5B_os/                      # 0.5B one-step checkpoint
├── VOSR_1.4B_ms/                      # 1.4B multi-step checkpoint
└── VOSR_1.4B_os/                      # 1.4B one-step checkpoint
```

代码中默认路径：

- SD2.1 VAE：`preset/ckpts/stable-diffusion-2-1-base`
- Qwen VAE：`preset/ckpts/Qwen-Image-vae-2d`
- SD2 轻量 decoder：`preset/ckpts/sd21_lwdecoder.pth`
- DINOv2 缓存：`preset/ckpts/torch_cache`

推理脚本会优先在 checkpoint 目录下寻找：

```text
clean_weights/ema_model.safetensors
clean_weights/model.safetensors
checkpoints/ema_model.safetensors
checkpoints/model.safetensors
ema_model.safetensors
model.safetensors
```

如果找不到，会递归搜索 checkpoint 目录下的 `.safetensors` 文件。

## 5. 数据配置

### 5.1 数据读取模式

训练配置通过 `dataset_type` 指定数据读取方式：

- `txt`：从 txt 列表读取单张图片路径。
- `webdataset`：从目录中的 `.tar` shard 读取图片。

配置入口在 YAML 的：

```yaml
train_dataset_config: configs/train_txt/train_dataset_txt.txt
dataset_type: 'webdataset'
```

### 5.2 `configs/train_txt/*.txt`

`train_dataset_config` 文件每一行格式为：

```text
<数据列表或数据目录路径>, <采样权重>
```

示例：

```text
/path/to/dataset_A, 2
/path/to/dataset_B, 1
```

在当前代码中，`train_vosr.py` 和 `train_vosr_distill.py` 会调用 `load_dataset_config()`，把该文件解析成：

- `args.train_dataset_txt_paths_list`
- `args.train_dataset_prob_paths_list`

如果 `dataset_type: txt`，这些路径应指向包含图片路径列表的 txt 文件。`TxtPairDataset` 会读取每个 txt 中的图片路径，并按采样权重重复加入样本列表。

如果 `dataset_type: webdataset`，这些路径应指向包含 `.tar` 文件的目录。`build_webdataset_pipeline()` 会在每个目录下查找 `*.tar`。

### 5.3 数据样本输出

`dataloaders/realsr_dataset.py` 中的数据集只输出 HQ 图像：

```python
{"hq": hq_tensor}
```

训练时的 LQ 图像不是从磁盘读取，而是在训练循环中通过 `RealESRGAN_degradation.degrade_process()` 在线生成。

### 5.4 裁剪策略

`RandomCropTransform`：

- 训练时使用。
- 如果图片短边小于 `args.resolution`，先按短边放大到目标尺寸。
- 随机裁剪出 `resolution x resolution` 图像块。

`CenterCropTestTransform`：

- 测试时使用。
- 同样先保证短边足够大，再中心裁剪。

## 6. 图像退化模块

退化逻辑在 `dataloaders/realesrgan_gpu.py`，配置文件为 `dataloaders/params_realsr.yml`。

`RealESRGAN_degradation` 会在 GPU 上对 HQ 图像执行 RealESRGAN 风格退化，主要步骤包括：

- 一阶或二阶 blur。
- 随机 resize，模式包括 `area`、`bilinear`、`bicubic`。
- 高斯噪声或泊松噪声。
- JPEG 压缩。
- sinc filter。
- 可选灰度化。
- 可选 color jitter。
- 最后 clamp/round 到 `[0, 1]`。

当前 `params_realsr.yml` 中：

```yaml
scale: 4
first_order_prob: 1
second_order_prob: 1
color_jitter_prob: 0.0
gray_prob: 0.0
```

训练脚本调用方式：

```python
degradation = RealESRGAN_degradation('params_realsr.yml', device=accelerator.device)
_, lq = degradation.degrade_process(hq, resize_bak=True)
```

`resize_bak=True` 表示退化后的 LQ 会重新 resize 回 HQ 的空间尺寸，之后再送入 VAE 编码。推理时也会先把输入图像 bicubic 放大到目标尺寸，再送入 VOSR。

## 7. 核心模型模块

### 7.1 `models/lightningdit.py`

这是主干生成模型，基于 DiT/SiT 风格 Transformer。

主要类：

- `LightningDiT`：整体 DiT 模型。
- `LightningDiTBlock`：Transformer block，包含自注意力、可选跨注意力、MLP 和 timestep 调制。
- `Attention`：自注意力模块。
- `MultiHeadCrossAttention`：用于融合 DINOv2 视觉特征的跨注意力。
- `TimestepEmbedder`：把连续时间步 `t` 或辅助时间步 `r` 编码成 hidden embedding。
- `FinalLayer`：最终把 token 输出还原为 latent patch。

`LightningDiT` 输入：

```python
x: [B, 2 * latent_channels, H, W]
t: [B]
r: [B] 或 None
z: DINOv2 特征列表或 None
```

其中 `x` 是 `lq_latent` 和当前噪声/状态 `z_t` 在 channel 维拼接后的结果。

`LightningDiT` 输出：

```python
[B, latent_channels, H, W]
```

它表示 flow velocity 或 shortcut velocity。

关键配置：

- `patch_size`：latent patch 大小。
- `dim`：Transformer hidden size。
- `depth`：block 数。
- `num_heads`：attention head 数。
- `use_qknorm`：是否启用 QK norm。
- `use_swiglu`：MLP 是否使用 SwiGLU。
- `use_rope`：是否使用 rotary position embedding。
- `use_rmsnorm`：是否用 RMSNorm。
- `auxiliary_time_cond`：one-step 蒸馏 student 会启用，用于处理 `r`。

`forward_flexible()` 用于推理：

- 支持不同输入分辨率。
- 如果开启 RoPE，当前推理尺寸与训练尺寸不一致时会动态生成 RoPE。
- 当前实现要求输入 latent 的 H 和 W 相等。

### 7.2 `models/light_decoder.py`

这是 SD2.1 VAE 的轻量 decoder 替代模块，仅用于 `ae_type: sd2`。

作用：

- 输入 `[B, 4, H/8, W/8]` latent。
- 输出 `[B, 3, H, W]` RGB 图像。
- 相比原始 SD2 decoder，通道数减半：
  - 原始 decoder block 通道约为 `[128, 256, 512, 512]`
  - 轻量 decoder 为 `[64, 128, 256, 256]`

推理脚本中，当 `args.ae_type == 'sd2'` 时，会从 `preset/ckpts/sd21_lwdecoder.pth` 加载该 decoder。

### 7.3 `models/qwenimage_vae2d.py`

这是 Qwen-Image VAE 的 2D 版本，供 `ae_type: qwen` 使用。

主要组件：

- `Encoder2D`：RGB 图像编码到 latent。
- `Decoder2D`：latent 解码回 RGB。
- `ResidualBlock2D`、`AttentionBlock2D`、`MidBlock2D`：VAE 内部结构。
- `AutoencoderKLQwenImage2D`：对外提供 `from_pretrained()`、`encode()`、`decode()` 等接口。

Qwen VAE 使用 16 通道 latent。训练和推理时会使用配置中的 `latents_mean` 与 `latents_std` 对 latent 做归一化/反归一化。

### 7.4 其他模型工具文件

- `models/rmsnorm.py`：RMSNorm 实现。
- `models/swiglu_ffn.py`：SwiGLU 前馈网络。
- `models/pos_embed.py`：RoPE、位置编码相关函数。
- `models/__init__.py`：模型包初始化文件。

## 8. `vosr.py` 核心逻辑

`vosr.py` 定义 `VOSR(nn.Module)`，负责训练 loss、CFG 条件构造和采样。

### 8.1 条件插值

```python
interpolate(cond, uncond, alpha, interp_type)
```

支持两种模式：

- `lin`：线性插值，`alpha * cond + (1 - alpha) * uncond`
- `sph`：球面风格插值，`alpha * cond + sqrt(1 - alpha^2) * uncond`

训练时会用它构造：

- 弱 LQ 条件。
- 加噪 LQ 条件。
- CFG 中的 conditional/unconditional 分支。

### 8.2 CFG 条件构造

`_prepare_cfg_conditions()` 用于 multi-step 训练：

- 随机选择一部分 batch 作为 CFG 弱条件样本，比例由 `cfg_ratio` 控制。
- `lq_noised` 使用 `cond_strength_aelq_list` 随机生成的强度。
- `lq_weak` 使用 `weak_cond_strength_aelq_list` 范围随机采样。
- DINOv2 特征 `z` 的 weak 分支置零。

`_prepare_cfg_conditions_distill()` 用于 one-step 蒸馏：

- 与上面类似，但 weak condition 使用标量 `args.weak_cond_strength_aelq`。

### 8.3 multi-step 训练 loss

```python
loss_fm(model, lq, hq, z)
```

流程：

1. 采样时间步 `t`。
2. 采样噪声 `eps`。
3. 构造 noisy latent：

   ```python
   z_t = (1 - t) * hq + t * eps
   ```

4. 目标 velocity：

   ```python
   v = eps - hq
   ```

5. 模型输入：

   ```python
   inp = concat(lq_mixed, z_t)
   ```

6. 计算 MSE：

   ```python
   loss = mean((model(inp, t, z=z_mixed) - v) ** 2)
   ```

### 8.4 one-step 蒸馏 loss

`loss_fm_distill_shortcut_improved()`：

- 用 teacher 或当前模型构造 CFG teacher target。
- 训练 student 预测从 `t` 到 `r` 的 shortcut velocity。
- 使用中点近似构造 integral target。

`loss_fm_distill_rcgm_improved()`：

- 同样先构造 teacher target。
- 额外调用 `_rcgm_consistency()` 做 RCGM consistency 约束。
- `rcgm_delta_t` 和 `rcgm_n_steps` 可从 args 读取；如果 YAML 没有设置，默认分别为 `0.01` 和 `2`。

### 8.5 推理采样

`sample_multistep_fm()`：

- multi-step 模型使用。
- 从标准高斯噪声开始。
- 默认线性时间表 `1 -> 0`。
- 每一步调用 DiT 预测 velocity，然后更新：

  ```python
  z = z - dt * d_cur
  ```

- 当当前时间步落在 `[t_start, t_end]` 内时启用 CFG：

  ```python
  d = d_weak + cfg_scale * (d_cond - d_weak)
  ```

`sample_onestep()`：

- one-step student 使用。
- 默认 `infer_steps=1`。
- 每一步输入 `t_cur` 和 `t_next`，模型通过辅助时间条件预测跨步 velocity。

## 9. 训练脚本：`train_vosr.py`

这是 multi-step 模型训练入口。

### 9.1 启动命令

0.5B：

```bash
torchrun --nproc_per_node=8 train_vosr.py --config configs/train_yml/multi_step/VOSR_0.5B.yml
```

1.4B：

```bash
torchrun --nproc_per_node=8 train_vosr.py --config configs/train_yml/multi_step/VOSR_1.4B.yml
```

### 9.2 主流程

1. 读取 YAML 配置。
2. 解析 `train_dataset_config`，得到训练数据路径和采样权重。
3. 根据配置自动拼接实验名 `args.exp_name`。
4. 创建 `Accelerator`，配置混合精度、梯度累积和日志。
5. 加载 DINOv2 视觉编码器。
6. 加载 VAE：
   - `sd2` 使用 `diffusers.AutoencoderKL`
   - `qwen` 使用 `AutoencoderKLQwenImage2D`
7. 创建 `LightningDiT`，`auxiliary_time_cond=False`。
8. 创建 AdamW 或 8-bit AdamW 优化器。
9. 创建学习率 scheduler。
10. 创建 `RealESRGAN_degradation`。
11. 自动恢复 checkpoint，或加载 `pretrained_ckpt`。
12. 创建训练 DataLoader。
13. 创建 `VOSR` loss 封装。
14. 可选创建 EMA 模型。
15. 进入训练循环。

### 9.3 每个 batch 的训练步骤

1. 从 dataloader 取出 HQ 图像。
2. 使用 RealESRGAN 退化生成 LQ。
3. 把 HQ/LQ 从 `[0, 1]` 转为 `[-1, 1]`。
4. 用 DINOv2 从 LQ 提取视觉特征。
5. 用 VAE 同时编码 LQ 和 HQ。
6. 调用 `vosr.loss_fm()` 计算 flow matching loss。
7. 反向传播、梯度裁剪、optimizer step。
8. 更新 EMA。
9. 按 `checkpointing_steps` 保存 checkpoint。
10. 按 `inference_steps` 运行验证推理并记录 LPIPS/MUSIQ。

### 9.4 输出文件

训练输出目录：

```text
<output_dir>/<exp_name>/
├── args.json                         # 当前训练配置快照
├── log.txt                           # 日志
├── checkpoints/
│   └── checkpoint-00010000/
│       ├── ...                       # accelerate 保存的训练状态
│       └── clean_weights/
│           ├── model.safetensors
│           └── ema_model.safetensors
└── sr/
    └── 00010001/                     # 验证阶段生成的 SR 图片
```

推理通常使用 `clean_weights/ema_model.safetensors`。

## 10. 蒸馏训练脚本：`train_vosr_distill.py`

这是 one-step student 的训练入口。

### 10.1 启动命令

0.5B：

```bash
torchrun --nproc_per_node=8 train_vosr_distill.py --config configs/train_yml/one_step/VOSR_0.5B.yml
```

1.4B：

```bash
torchrun --nproc_per_node=8 train_vosr_distill.py --config configs/train_yml/one_step/VOSR_1.4B.yml
```

### 10.2 与 multi-step 训练的区别

蒸馏训练会创建两个 DiT：

- teacher：`auxiliary_time_cond=False`，加载 multi-step checkpoint，冻结参数。
- student：`auxiliary_time_cond=True`，训练 one-step/shortcut 预测能力。

关键 YAML 参数：

```yaml
teacher_ckpt: preset/ckpts/VOSR_0.5B_ms/checkpoints/ema_model.safetensors
pretrained_ckpt: preset/ckpts/VOSR_0.5B_ms/checkpoints/ema_model.safetensors
distill_type: rcgm
infer_steps: 1
u_weight: 1.0
```

`distill_type` 支持：

- `shortcut`
- `rcgm`

训练循环大体与 `train_vosr.py` 一致，但 loss 调用改为：

```python
vosr.loss_fm_distill_shortcut_improved(...)
```

或：

```python
vosr.loss_fm_distill_rcgm_improved(...)
```

## 11. 推理脚本：multi-step

入口文件为 `inference_vosr.py`。

### 11.1 基本命令

0.5B multi-step：

```bash
python inference_vosr.py \
    -c preset/ckpts/VOSR_0.5B_ms \
    -i preset/datasets/inp_data \
    -o preset/results \
    -u 4
```

1.4B multi-step：

```bash
python inference_vosr.py \
    -c preset/ckpts/VOSR_1.4B_ms \
    -i preset/datasets/inp_data \
    -o preset/results \
    -u 4
```

### 11.2 常用参数

```text
-c, --checkpoint       checkpoint 目录
-i, --input_image      输入图片或输入文件夹
-o, --output_dir       输出目录
-u, --upscale          放大倍数，默认 4
--infer_steps          推理步数，默认 25
--cfg_scale            CFG scale，默认 2.0；可覆盖 args.json
--weak_cond_strength_aelq  弱条件强度，默认 0.10
--align_method         颜色对齐方式：adain、wavelet、nofix
--tile_size            tile 推理尺寸，0 表示关闭
--tile_overlap         tile 重叠大小
--seed                 随机种子
--force_rerun          即使输出已存在也重新推理
```

### 11.3 推理流程

1. 从 checkpoint 附近读取 `args.json`。
2. CLI 中显式传入的参数覆盖 `args.json`。
3. 枚举输入图片，支持：
   - `.png`
   - `.jpg`
   - `.jpeg`
   - `.bmp`
   - `.tif`
   - `.tiff`
   - `.webp`
4. 创建输出子目录：

   ```text
   <output_dir>/<ae_type>_steps<infer_steps>_cfg<cfg_scale>_wc<weak_cond_strength_aelq>/
   ```

5. 如果输出已全部存在，默认跳过；可用 `--force_rerun` 强制重跑。
6. 加载 VAE、LightDecoder、DINOv2、LightningDiT 权重。
7. 读取输入图片并 bicubic 放大到目标尺寸。
8. VAE 编码 LQ。
9. DINOv2 提取 LQ 视觉特征。
10. 调用 `vosr_model.sample_multistep_fm()` 采样 SR latent。
11. VAE/LightDecoder 解码到图像。
12. 可选执行颜色对齐。
13. 保存结果。

### 11.4 tile 推理

当 `--tile_size > 0` 时，脚本使用 latent-space tiled inference。

特点：

- 整图先 VAE encode 一次，避免 encoder tile 边界问题。
- 在 latent 空间切 tile，只对 DiT 前向做切块。
- 每个 tile 独立提取 DINOv2 特征。
- 重叠区域使用 Gaussian 权重融合 velocity。
- 最后整图 VAE decode 一次。

适用场景：

- 输入图片较大，显存不足。
- 需要降低单次 DiT 前向的空间尺寸。

## 12. 推理脚本：one-step

入口文件为 `inference_vosr_onestep.py`。

### 12.1 基本命令

0.5B one-step：

```bash
python inference_vosr_onestep.py \
    -c preset/ckpts/VOSR_0.5B_os \
    -i preset/datasets/inp_data \
    -o preset/results \
    -u 4
```

1.4B one-step：

```bash
python inference_vosr_onestep.py \
    -c preset/ckpts/VOSR_1.4B_os \
    -i preset/datasets/inp_data \
    -o preset/results \
    -u 4
```

### 12.2 与 multi-step 推理的区别

- 默认 `--infer_steps=1`。
- `LightningDiT` 构建时 `auxiliary_time_cond=True`。
- 采样函数为 `vosr_model.sample_onestep()`。
- 输出子目录格式为：

  ```text
  <output_dir>/<ae_type>_steps<infer_steps>_seed<seed>_<distill_type>/
  ```

one-step 推理也支持 tile 推理，逻辑与 multi-step 类似，但不使用 CFG 分支。

configs/train_yml/

### 13.1 数据和评估配置

```yaml
train_dataset_config: configs/train_txt/train_dataset_txt.txt
dataset_type: 'webdataset'
shuffle_buffer: 2000
test_lq_dir: preset/datasets/RealSR/test_LR
test_gt_dir: preset/datasets/RealSR/test_HR
test_upscale: 4
```

作用：

- `train_dataset_config`：训练数据路径和权重配置。
- `dataset_type`：选择 `txt` 或 `webdataset`。
- `shuffle_buffer`：WebDataset shuffle buffer。
- `test_lq_dir`/`test_gt_dir`：训练中定期评估使用。
- `test_upscale`：评估时是否先放大 LQ。

### 13.2 模型结构配置

0.5B 典型配置：

```yaml
ae_type: sd2
dim: 1024
depth: 28
num_heads: 16
enc_type: dinov2b
enc_dim: 768
layer_dinov2b_list: [8]
```

1.4B 典型配置：

```yaml
ae_type: qwen
dim: 1536
depth: 36
num_heads: 24
enc_type: dinov2l
enc_dim: 1024
layer_dinov2b_list: [17]
```

通用结构参数：

- `resolution`：训练裁剪分辨率。
- `patch_size`：latent patch 大小。
- `mlp_ratio`：Transformer MLP 扩展比例。
- `use_qknorm`：是否使用 QK norm。
- `use_swiglu`：是否使用 SwiGLU。
- `use_rope`：是否使用 RoPE。
- `use_rmsnorm`：是否使用 RMSNorm。
- `encdim_ratio`：DINOv2 特征映射 MLP 的 hidden 扩展比例。

### 13.3 训练超参数

```yaml
learning_rate: 5.0e-5
max_train_steps: 400001
checkpointing_steps: 10000
inference_steps: 10000
train_batch_size: 8
gradient_accumulation_steps: 1
mixed_precision: bf16
use_ema: True
ema_decay: 0.9999
```

说明：

- `train_batch_size` 在代码中按全局 batch size 理解，会除以 GPU 数得到单卡 batch。
- `checkpointing_steps` 控制保存间隔。
- `inference_steps` 控制训练中评估间隔。
- `mixed_precision` 通常为 `bf16`。

### 13.4 CFG 和条件强度

```yaml
cfg_ratio: 0.1
cfg_scale: -0.5
weak_cond_strength_aelq_list: [0.05, 0.25]
cond_strength_aelq_list: [5, 1]
interp_type: lin
t_start: 0
t_end: 1
```

说明：

- `cfg_ratio`：训练时随机转为 weak condition 的样本比例。
- `cfg_scale`：推理和 teacher target 中的 CFG 强度。
- `weak_cond_strength_aelq_list`：multi-step 训练和推理中 weak LQ 条件范围。
- `weak_cond_strength_aelq`：one-step 蒸馏使用的固定 weak condition 强度。
- `cond_strength_aelq_list`：训练时 LQ 加噪条件强度的分布参数。
- `t_start`/`t_end`：CFG 生效的时间范围。

### 13.5 蒸馏配置

```yaml
teacher_ckpt: preset/ckpts/VOSR_1.4B_ms/checkpoints/ema_model.safetensors
pretrained_ckpt: preset/ckpts/VOSR_1.4B_ms/checkpoints/ema_model.safetensors
distill_type: rcgm
u_weight: 1.0
infer_steps: 1
```

说明：

- `teacher_ckpt`：teacher multi-step 权重。
- `pretrained_ckpt`：student 初始化权重。
- `distill_type`：蒸馏 loss 类型。
- `u_weight`：shortcut/consistency loss 权重。

## 15. 训练与推理中的数据流

### 15.1 multi-step 训练数据流

```text
HQ image
  -> RandomCrop
  -> RealESRGAN degradation
  -> LQ image
  -> DINOv2 提取视觉特征 z
  -> VAE encode LQ/HQ
  -> 构造 z_t = (1 - t) * HQ_latent + t * noise
  -> concat(LQ_latent_condition, z_t)
  -> LightningDiT
  -> flow matching loss
```

### 15.2 one-step 蒸馏数据流

```text
HQ image
  -> RealESRGAN degradation 得到 LQ
  -> DINOv2 提取视觉特征
  -> VAE encode
  -> teacher multi-step 模型产生 CFG target
  -> student 预测 shortcut velocity
  -> shortcut 或 RCGM consistency loss
```

### 15.3 推理数据流

```text
Input image
  -> bicubic resize 到目标放大尺寸
  -> VAE encode
  -> DINOv2 提取视觉特征
  -> VOSR sampling
  -> VAE / LightDecoder decode
  -> AdaIN 或 wavelet 颜色对齐
  -> 保存输出图片
```

## 16. 注意事项

1. `torch.hub.set_dir('preset/ckpts/torch_cache')` 会让 DINOv2 通过 torch hub 使用该缓存目录。
2. 如果本地没有 DINOv2 缓存，首次加载可能需要联网下载。
3. `train_batch_size` 在训练脚本中按全局 batch 处理，单卡 batch 为 `train_batch_size // accelerator.num_processes`。
4. `LightningDiT.forward_flexible()` 当前要求 latent 输入是方形。
5. 推理脚本会读取 checkpoint 目录附近的 `args.json`，命令行参数只有显式传入时才覆盖它。
6. 输出已存在时推理会跳过，可使用 `--force_rerun` 或设置环境变量 `INFER_NO_SKIP=1`。
7. 0.5B 使用 SD2 latent 4 通道，1.4B 使用 Qwen latent 16 通道，二者 checkpoint 和配置不能混用。
8. one-step checkpoint 需要使用 `inference_vosr_onestep.py`，multi-step checkpoint 需要使用 `inference_vosr.py`。
9. 训练保存的 accelerator checkpoint 用于继续训练，`clean_weights/*.safetensors` 更适合推理或作为预训练权重。
