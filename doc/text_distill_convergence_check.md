# 文本蒸馏消融实验收敛性检查

检查对象为 `doc/text_distill_ablation_commands.md` 中的 A/B/C 三个 20k 蒸馏实验。

## 结论

从现有日志和 TensorBoard 标量看，**不能把 A/B/C 当前效果简单归因于“训练步数太少、完全没收敛”**。三组都完整跑到了 `checkpoint-00020000`，并且 5 月 9 日的重新启动只是从 20k checkpoint 恢复，配置总步数仍是 20k，基本没有继续训练。

但也不能证明 20k 已经是最优点。原因是当前训练日志只记录训练 loss，没有记录中间 checkpoint 的真实验证指标；训练 loss 又强烈受 batch、时间步 `t/r`、目标构造影响，不能直接代表 OCR-EM、CER、NIQE 等最终指标。

更准确的判断是：

- A/B：loss 在 20k 内没有持续下降趋势，继续训练未必会带来显著收益。
- C：总 loss 仍然波动较大，但它的最终 OCR 指标已经差于 B；这更像是当前 RCGM/RC 一致性约束和文本识别目标不完全一致，而不只是训练步数不足。
- 若要验证“20k 不够”，最有效的是补做 B/C 的 `10k/15k/20k` 甚至 `30k/40k` checkpoint 推理评估，而不是只看训练 loss。

| B | guided target, no RC | `00001000` 到 `00020000` 共 20 个 | `loss/lr/v_loss` | 完整完成 20k |
| C | guided target + RC/RCGM | `00001000` 到 `00020000` 共 20 个 | `loss/lr/v_loss` | 完整完成 20k |

注意：当前代码中 `train_vosr_distill.py` 把 `v_loss` 记录成了 `loss.item()`。因此：

- A/B 中 `v_loss == loss` 是合理的，因为 `u_weight=0`。
- C 中日志里的 `v_loss` 实际是总 loss，不是单独的 teacher-student velocity loss。
- 因此 C 的日志不能直接分解出 `v_loss` 和 `u_loss/rc_loss` 各自的变化。

## 对“训练步数太少”的判断

### A/B

A/B 的训练 loss 在 20k 内没有表现为“还在稳定下降”。A 甚至从前 1k 到后 20k 的窗口均值略升，B 则基本在 `1.3e-3 - 1.5e-3` 附近波动。这个现象不一定说明模型变差，因为 batch 难度和采样时间步会导致 loss 噪声，但至少不支持“只要继续训练，loss 明显还会持续降”的判断。

结合最终指标，B 已经在 OCR-EM、OCR-CER、OCR-CharAcc、MANIQA 上优于 A 和蒸馏前 teacher。因此 B 的 20k 结果更像是当前设置已经形成了有效蒸馏，而不是明显欠训练。

### C

C 的 total loss 明显高于 A/B，是因为它包含 RCGM/RC 一致性相关项，不能和 A/B 的 loss 数值直接比较。C 的窗口均值从前 1k 的 `0.505` 降到后 19k-20k 的 `0.373`，说明它确实还在学习这个一致性约束；但最终 OCR 指标比 B 更差：

| 指标 | B guided no RC | C guided + RC | 方向 |
| --- | ---: | ---: | --- |
| OCR-EM | 0.5841 | 0.5548 | B 更好 |
| OCR-CER | 0.3401 | 0.3599 | B 更好 |
| OCR-CharAcc | 0.6599 | 0.6401 | B 更好 |
| NIQE | 10.9392 | 6.5167 | C 更好 |
| DISTS | 0.2200 | 0.2078 | C 更好 |

这说明 C 的 RC/RCGM 项可能更偏向自然图像质量、分布一致性或平滑性，而不一定直接服务于文本可识别性。它不是简单的“训练太短所以 OCR 差”；更可能是目标函数方向和 OCR 目标存在偏差。

## 建议下一步

1. 先不要急着把 A/B/C 全部延长训练。
2. 优先对 B 做 `10k/15k/20k` checkpoint 推理，确认 OCR 指标是否已经在 20k 附近平台化。
3. 对 C 只需要补测 `10k/15k/20k`，判断 OCR 是否早期更好、后期被 RC 项拉偏。
4. 如果要继续训练，优先延长 B 到 `40k` 作为“步数是否不足”的对照；C 不建议作为第一优先级。
5. 后续最好修改日志，把 `v_loss`、`u_loss/rc_loss`、`repa_loss` 分开记录，否则很难判断是哪一项在主导训练。
